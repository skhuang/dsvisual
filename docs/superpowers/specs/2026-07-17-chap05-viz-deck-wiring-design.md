## Motivation & the lint gate

ds2026 lecture decks reference dsvisual visualizations with a `<!-- viz: ID "TITLE" -->`
directive (own line), which the pipeline expands to a blockquote link
`> ‚Ė∂ [šļíŚčēŤ¶ĖŤ¶ļŚĆĖ:TITLE](https://skhuang.github.io/dsvisual/#m=ID)`. `chap05_trees_core.md`
currently has **none**, although 12 viz map to its topics.

dsvisual's consistency lint (`pipeline/lint.js`) treats a viz ID as valid only if a
`slides/en/<ID>.md` file exists ‚ÄĒ and those files are generated by `build_slides.js`
from **`slides_db.js`**. The 7 existing tree viz have `slides_db` entries; the **5 new**
ones (`tree-array-rep`, `tree-reconstruct`, `tree-catalan`, `decision-tree-coins`,
`tree-copy-equal`) do not. So wiring the new viz requires first giving them `slides_db`
entries.
