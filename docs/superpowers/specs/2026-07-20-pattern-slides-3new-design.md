# Bilingual slide decks for the 3 new design patterns ‚Äî design

- Date: 2026-07-20
- Repo: `/Users/skhuang/course/dsvisual`
- Follow-up to the patterns-registry refactor (PR #147), which added 3 new patterns
  (Builder, Command, Composite) without slide decks.

## Problem

The 11 pre-existing design patterns each have a bilingual (zh/en) explainer deck, defined
in `slides_db.js` (as object keys `"pattern-<id>": {‚Ä¶}`) and generated by
`build_slides.js` into `slides/{en,zh}/pattern-<id>.md` + `js/slides_rendered.js` (consumed
by the site's per-method **"Slides"** button). The 3 new patterns ‚Äî **`pattern-builder`,
`pattern-command`, `pattern-composite`** ‚Äî have **no `slides_db.js` entries**, so no decks:
their "Slides" button has nothing to show, and they can't be ds2026 deck-wired (the
`pipeline/lint.js` `discoverVizIds` gate needs `slides/en/<id>.md`).

## Decision (approved)

Add 3 bilingual `slides_db.js` entries matching the existing pattern-deck structure, then
regenerate. Same recipe as the chap05 slides_db work (PR #140). Source code excerpts reuse
the `cpp/pattern_{builder,command,composite}.cpp` authored in the refactor.

## Deck structure (mirror the existing pattern decks ‚Äî 8 slides)

Each entry: `"pattern-<id>": { category: "Design Patterns", title: {zh,en}, slides: [...] }`
with these 8 slides (headings bilingual), using the established block vocabulary:

1. **`<Pattern> Pattern`** ‚Äî `paragraph`: one-sentence GoF intent (name the category).
2. **Core Concept** ‚Äî `paragraph` + `bullets`: participants and their roles.
3. **Operation Flow** ‚Äî `steps` + a **`mermaid`** block (`{type:'mermaid', code:'flowchart LR\n‚Ä¶'}`) showing the runtime flow. Mermaid is auto-rendered by `mmdc` at build; the `code` is a single source string (not translated).
4. **UML Structure Diagram** ‚Äî a hand-drawn **`svg`** block (`{type:'svg', svg:'<svg ‚Ä¶>‚Ä¶</svg>'}`, not translated) + a `note` (`{text:{zh,en}}`).
5. **Pattern Properties** ‚Äî a **`table`** (`{headers:[{zh,en}], rows:[[{zh,en},‚Ä¶]]}`) with GoF Category / Participants / Intent / (pattern-specific) / (pattern-specific) + a **`math`** block (`{tex, caption:{zh,en}}`) capturing the pattern's essence.
6. **Source Code** ‚Äî a **`code`** block (`{lang:'cpp', code:'‚Ä¶'}`) ‚Äî the illustrative C++ from `cpp/pattern_<id>.cpp` (trimmed to a focused excerpt like the existing decks).
7. **Pros, Cons & When to Use** ‚Äî `bullets`.
8. **Summary** ‚Äî `bullets`.

Block-type shapes (verbatim from existing pattern decks): text leaves are `{zh,en}`;
`mermaid.code`, `svg.svg`, `code.code`, `math.tex` are single (untranslated) strings;
`math.caption` and `note.text` are `{zh,en}`; `table.headers`/`rows` cells are `{zh,en}`.

## Per-pattern content (GoF-accurate)

- **`pattern-builder`** (Creational) ‚Äî participants Director / Builder (interface) /
  ConcreteBuilder / Product; flow: Director drives `buildPartA/B` ‚Üí `getResult()`; math e.g.
  `\text{Product} = b.\text{buildA}() \circ b.\text{buildB}() \circ b.\text{getResult}()`;
  properties GoF=Creational, Participants=Director/Builder/ConcreteBuilder/Product,
  Intent=separate construction of a complex object from its representation.
- **`pattern-command`** (Behavioral) ‚Äî Invoker / Command (interface) / ConcreteCommand /
  Receiver; flow: `invoker.run()` ‚Üí `command.execute()` ‚Üí `receiver.action()` (+ undo via
  history); Intent=encapsulate a request as an object (queue/log/undo).
- **`pattern-composite`** (Structural) ‚Äî Component (interface) / Leaf / Composite (holds
  children); flow: `Composite.operation()` recurses into children, `Leaf.operation()` does
  work; Intent=treat individual objects and compositions uniformly (whole-part tree).

## Files & build

- Modify: `slides_db.js` (add the 3 entries in the Design-Patterns region, near the other
  `"pattern-*"` keys).
- Run `npm run build:slides` ‚Äî renders mermaid via `mmdc` (node_modules; puppeteer config
  already present) and regenerates: `slides/en/pattern-{builder,command,composite}.md`,
  `slides/zh/pattern-{builder,command,composite}.md`, and `js/slides_rendered.js`.
- **Do NOT hand-edit** the generated `slides/**` or `js/slides_rendered.js` ‚Äî only
  `slides_db.js` is source.
