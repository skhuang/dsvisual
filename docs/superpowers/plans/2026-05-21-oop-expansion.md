## Background the engineer must know

dsvisual is a browser-based C++ data-structure/algorithm visualizer opened from `file://`. Key facts about the OOP category:

- **`app.js`** is one large file wrapped in `document.addEventListener('DOMContentLoaded', () => { ... })`. All functions/consts named below live inside that closure.
- **`METHOD_GROUPS`** (~line 96) drives the category nav + method dropdown. The `oop` group (id `'oop'`, ~line 191) currently ends with the `oop-encapsulation` entry.
- **`getCodeForMethod`** (~line 214) maps a method id to its `code*` constant from `code_db.js`. The OOP entries are at ~lines 274-276.
- **`code_db.js`** is auto-generated by `build_db.js` from `.cpp` files via an explicit `mappings` table. **The mapping MUST exist before regeneration**, or the constant is silently dropped.
- **OOP visualizer mechanism:** `#oop-container` > `#oop-visualization` holds one static `<div id="oop-*-view" class="oop-view hidden">` per method, each containing an `<h3>`, an `<svg id="oop-*-svg">`, and a legend `<div>`. The dropdown `#oop-mode-select` (JS const `oopModeSelect`) chooses the active view. The JS consts `oopInheritanceView`, `oopPolymorphismView`, `oopEncapsulationView` (~lines 783-785) reference the view divs.
- **`updateLayout()`** (~line 1582) is a big `if/else` on `currentMode`. Its `currentMode.includes('oop-')` branch (~lines 1731-1755) shows `oopContainer` + `oopActions`, hides all `oop-*-view` divs, then a per-method `else if` un-hides one view, sets `codeTitle`/`codeDisplay`, and sets `oopModeSelect.value`.
- **`renderAll()`** (~line 1773) dispatches `currentMode.includes('oop-')` → `renderOOP()`. `renderOOP()` (~line 3579) reads `oopModeSelect.value` and calls `renderOOPInheritance()` / `renderOOPPolymorphism()` / `renderOOPEncapsulation()`.
- **The "Visualize Concept" button** (`#btn-oop-demo`, JS const `btnOopDemo`): its click handler (~line 3868) does `currentMode = 'oop-' + oopModeSelect.value; updateLayout(); renderAll();` then an `if/else` that runs an extra animation for the 3 existing modes. For the new modes the `if/else` simply does not match — clicking the button still re-renders the diagram via the `renderAll()` call. **No change to the `btn-oop-demo` handler is needed.**
- **Render style:** existing render functions (e.g. `renderOOPInheritance`, ~line 3586) draw the SVG with `document.createElementNS`. The new render functions use a shared `drawOopBox()` helper (introduced in Task 2) to stay concise.
- **Slide pipeline:** `slides_db.js` (a `SLIDES_DB` map of method-id → deck) → `npm run build:slides` → `slides_rendered.js` + `slides/{zh,en}/*.md`. `slides_db.js` is in canonical (expanded) form. `npm run format:code` clang-formats `.cpp` files and the C++ embedded in decks, then rebuilds slides — run it at the end and commit its output to stay idempotent.

Playwright tests live in `tests/visualizer.spec.js`; helper `loadMethod(page, methodId)` navigates within one page session. The OOP tests are at ~lines 270-289.

```cpp
#include <iostream>
#include <vector>
using namespace std;

// Abstract base class — a pure-virtual class is C++'s "interface".
class Shape {
public:
    virtual double area() const = 0;   // pure virtual -> Shape is abstract
    virtual ~Shape() {}
};

class Circle : public Shape {
    double r;
public:
    Circle(double radius) : r(radius) {}
    double area() const override { return 3.14159 * r * r; }
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(double width, double height) : w(width), h(height) {}
    double area() const override { return w * h; }
};

int main() {
    // Shape s;   // compile error: cannot instantiate an abstract class
    vector<Shape*> shapes = { new Circle(2.0), new Rectangle(3.0, 4.0) };
    for (Shape* s : shapes) {
        cout << "area = " << s->area() << endl;
        delete s;
    }
    return 0;
}
```

#include <iostream>
#include <string>
using namespace std;

// Function overloading: same name, different parameter types.
void print(int x) { cout << "int: " << x << endl; }
void print(double x) { cout << "double: " << x << endl; }
void print(const string& x) { cout << "string: " << x << endl; }

// Operator overloading.
struct Vector2D {
    double x, y;
    Vector2D operator+(const Vector2D& o) const {
        return Vector2D{ x + o.x, y + o.y };
    }
};

int main() {
    print(42);            // resolves to print(int)
    print(3.14);          // resolves to print(double)
    print(string("hi"));  // resolves to print(const string&)

    Vector2D a{1, 2}, b{3, 4};
    Vector2D c = a + b;   // operator+
    cout << "sum: (" << c.x << ", " << c.y << ")" << endl;
    return 0;
}
```

#include <iostream>
#include <string>
using namespace std;

// Function template.
template <typename T>
T maximum(T a, T b) {
    return (a > b) ? a : b;
}

// Class template.
template <typename T>
class Box {
    T value;
public:
    Box(T v) : value(v) {}
    T get() const { return value; }
    void set(T v) { value = v; }
};

int main() {
    cout << maximum(3, 7) << endl;                            // T = int
    cout << maximum(2.5, 1.5) << endl;                        // T = double
    cout << maximum(string("apple"), string("pear")) << endl; // T = string

    Box<int> bi(42);
    Box<string> bs("hello");
    cout << bi.get() << " " << bs.get() << endl;
    return 0;
}
```

```js
    function oopSvgEl(tag, attrs) {
        const el = document.createElementNS('http://www.w3.org/2000/svg', tag);
        for (const k in attrs) el.setAttribute(k, String(attrs[k]));
        return el;
    }
    // Draws a class box: a rect + a bold title + member-line texts. opts:
    // { x, y, w, h, title, titleColor, lines:[{text,color}], dashed }
    function drawOopBox(svg, opts) {
        const rect = oopSvgEl('rect', { x: opts.x, y: opts.y, width: opts.w, height: opts.h, class: 'oop-class-rect' });
        if (opts.dashed) rect.setAttribute('stroke-dasharray', '6 4');
        svg.appendChild(rect);
        const cx = opts.x + opts.w / 2;
        const title = oopSvgEl('text', {
            x: cx, y: opts.y + 22, 'text-anchor': 'middle', class: 'oop-member-text',
            style: 'font-weight:bold;fill:' + (opts.titleColor || '#60a5fa') + ';' + (opts.dashed ? 'font-style:italic;' : ''),
        });
        title.textContent = opts.title;
        svg.appendChild(title);
        (opts.lines || []).forEach((ln, i) => {
            const t = oopSvgEl('text', {
                x: cx, y: opts.y + 44 + i * 17, 'text-anchor': 'middle', class: 'oop-member-text',
                style: 'font-size:11px;fill:' + (ln.color || '#cbd5e1') + ';',
            });
            t.textContent = ln.text;
            svg.appendChild(t);
        });
    }
    function drawOopLabel(svg, x, y, text, color) {
        const t = oopSvgEl('text', {
            x: x, y: y, 'text-anchor': 'middle', class: 'oop-member-text',
            style: 'font-size:11px;fill:' + (color || '#cbd5e1') + ';',
        });
        t.textContent = text;
        svg.appendChild(t);
    }
    function drawOopLine(svg, x1, y1, x2, y2) {
        svg.appendChild(oopSvgEl('line', { x1: x1, y1: y1, x2: x2, y2: y2, class: 'oop-inheritance-line' }));
    }
```
