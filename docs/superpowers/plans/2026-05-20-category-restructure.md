# Category Restructure Implementation Plan (Spec 1 of 4)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure the app's 6 method categories into 9 more cohesive ones (Linear Structures, Trees, Graphs, Hash & Probabilistic, Heaps / Priority Queues, Sorting, Searching & String Matching, OOP Concepts, Design Patterns) without adding any new visualizers, animations, or slide content.

**Architecture:** Pure restructure. Rewrite `METHOD_GROUPS` in `app.js`; rewrite each slide deck's `category` string in `slides_db.js`; rebuild slides; update Playwright test assertions that referenced old category titles. No new files, no logic changes, no animation/CSS theme changes. Method IDs preserved unchanged so `code_db.js`, `slides_rendered.js` and `slides/{zh,en}/*.md` keep their semantic mappings.

**Tech Stack:** Vanilla JS (CommonJS), `node:test`, Playwright. Reuses existing `build_slides.js` pipeline; no new dependencies.

**Spec:** `docs/superpowers/specs/2026-05-20-category-restructure-design.md`

```bash
cat > /tmp/update_categories.js <<'EOF'
const fs = require('fs');
const path = require('path');
const dbPath = '/Users/skhuang/course/dsvisual/slides_db.js';
delete require.cache[require.resolve(dbPath)];
const db = require(dbPath);

const prefixMap = [
  [/^(stack-|queue$|list-)/, 'Linear Structures'],
  [/^tree-/, 'Trees'],
  [/^graph/, 'Graphs'],
  [/^hash-/, 'Hash & Probabilistic'],
  [/^heap-/, 'Heaps / Priority Queues'],
  [/^sort-/, 'Sorting'],
  [/^search-/, 'Searching & String Matching'],
  [/^oop-/, 'OOP Concepts'],
  [/^pattern-/, 'Design Patterns'],
];

for (const id of Object.keys(db)) {
  let matched = false;
  for (const [re, label] of prefixMap) {
    if (re.test(id)) { db[id].category = label; matched = true; break; }
  }
  if (!matched) throw new Error('No category for id ' + id);
}

// Match the canonical writer format used by format_code.js's formatSlidesDbCodeBlocksAt.
const body = JSON.stringify(db, null, 2);
const out = '// Structured slide content source. Hand-written.\n' +
            '// Consumed only by build_slides.js (Node). Do NOT load this in the browser.\n' +
            '// Text leaves are { zh, en } and may embed inline $...$ LaTeX.\n\n' +
            'const SLIDES_DB = ' + body + ';\n\n' +
            'module.exports = SLIDES_DB;\n';
fs.writeFileSync(dbPath, out);

const counts = {};
for (const k of Object.keys(db)) counts[db[k].category] = (counts[db[k].category] || 0) + 1;
console.log('Updated', Object.keys(db).length, 'decks. New category counts:', counts);
EOF
```
