## Fenwick Tree

A Fenwick tree (Binary Indexed Tree) supports point updates and prefix-sum queries, both in $O(\log n)$, using a single array and the bit trick `i & -i`.

---

## Core Concept

The array is 1-indexed; slot `i` stores the sum of the range `(i - lowbit(i), i]`, where `lowbit(i) = i & -i` is the lowest set bit.

- A prefix sum walks down via `i -= i & -i`.
- A point update walks up via `i += i & -i`.
- Far simpler and lower-constant than a segment tree, but only for invertible aggregates like sum.

---

## Operation Flow

1. `prefixSum(i)` starts at `i`, adds `bit[i]`, then jumps to `i - lowbit(i)`, repeating until 0.
2. `update(i, delta)` adds `delta` to `bit[i]`, then jumps to `i + lowbit(i)`, repeating past `n`.

## Complexity Analysis

| Operation | Complexity |
| --- | --- |
| point update | $O(\log n)$ |
| prefix sum | $O(\log n)$ |
| build | $O(n \log n)$ |
| space | $O(n)$ |

$$T(n) = O(\log n)$$

Each jump clears one set bit of `i`, so at most $\log n$ jumps.

---

## Source Code

```cpp
// add delta at 1-indexed position i, walking up via i += i & -i
void update(int i, long long delta) {
    for (; i <= n; i += i & -i)
        bit[i] += delta;
}

// sum of [1, i], walking down via i -= i & -i
long long prefixSum(int i) {
    long long s = 0;
    for (; i > 0; i -= i & -i)
        s += bit[i];
    return s;
}
```

## Pros, Cons & When to Use

- Pro: tiny, fast, easy to implement.
- Con: only invertible aggregates (sum), no native range update.
- Use: prefix sums / inversion counting / frequency tables where a segment tree would be overkill.

---

## Summary

- One array plus the `i & -i` trick.
- Walk down for a prefix sum, up for an update.
- Both ways are $O(\log n)$.
