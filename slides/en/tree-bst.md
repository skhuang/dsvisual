## Binary Search Tree (BST)

A Binary Search Tree is a binary tree where every node satisfies: all values in the left subtree < node value < all values in the right subtree, enabling $O(\log N)$ search, insert, and delete on average.

---

## Core Concept

The BST ordering property acts as a routing rule: during search, each comparison directs traversal left or right, reaching the target within at most $O(h)$ steps where $h$ is the tree height.

- Insert: recursively compare from root; insert a new node at the first null child.
- Search: go left if smaller, right if larger, equal means found.
- Worst case: inserting in sorted order degenerates into a linked list — $O(N)$ search.
- In-order traversal yields a sorted sequence.

---

## Operation Flow

1. Start at `root`; compare the value to insert with the current node.
2. If value < node value recurse into the left subtree; otherwise recurse right.
3. On encountering `nullptr`, allocate a new node here and return it.
4. Return up the call stack, updating child pointers at each level to complete the insertion.

## Complexity Analysis

| Operation | Time (avg) | Time (worst) |
| --- | --- | --- |
| search | $O(\log N)$ | $O(N)$ |
| insert | $O(\log N)$ | $O(N)$ |
| delete | $O(\log N)$ | $O(N)$ |
| Total Space | — | $O(N)$ |

$$T_{\text{search}}(n) = O(h),\quad h = O(\log N)\text{ avg},\; O(N)\text{ worst}$$

Average tree height is $O(\log N)$; degrades to $O(N)$ when inserting a sorted sequence.

## Source Code

```cpp
struct Node {
    int data;
    Node* left;
    Node* right;
    Node(int val) : data(val), left(nullptr), right(nullptr) {}
};

// Recursive insert returns updated subtree root
Node* insert(Node* node, int data) {
    if (!node) return new Node(data);
    if (data < node->data)
        node->left = insert(node->left, data);
    else if (data > node->data)
        node->right = insert(node->right, data);
    return node; // duplicate ignored
}

// In-order: left -> root -> right (sorted output)
void inorder(Node* node) {
    if (!node) return;
    inorder(node->left);
    cout << node->data << " ";
    inorder(node->right);
}
```

## Pros, Cons & When to Use

- Pro: simple to implement; average $O(\log N)$ search/insert/delete.
- Pro: in-order traversal produces a sorted sequence — good for range queries.
- Con: no self-balancing — worst case degrades to $O(N)$ linked list.
- Use when data is randomly distributed and strict balancing is not required (small ordered dictionaries).

---

## Summary

- BST ordering property (left < node < right) enables efficient routing.
- Average $O(\log N)$, worst $O(N)$; imbalance is the main risk.
- Self-balancing variants (AVL, Red-Black Tree) solve the imbalance problem.
