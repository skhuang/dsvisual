## Bloom Filter

A Bloom filter is a highly space-efficient probabilistic set used to test set membership. It may report a false positive but never a false negative.

---

## Core Concept

The structure is an $m$-bit array plus $k$ independent hash functions. Insertion sets the $k$ hashed bits to 1; a query checks whether those $k$ bits are all 1.

- All $k$ bits are 1 → "possibly present" — other elements may have set those bits.
- Any bit is 0 → "definitely not present".
- No deletion: clearing a bit would corrupt other elements sharing it.

---

## Operation Flow

1. Insert $x$: compute $h_1(x)$, $h_2(x)$, $h_3(x)$ and set those 3 bits to 1.
2. Query $y$: compute the 3 hashed positions of $y$.
3. All 3 bits 1 → "possibly present"; any bit 0 → "definitely not present".

## Complexity Analysis

| Aspect | Complexity |
| --- | --- |
| insert / query time | $O(k)$ |
| space | $O(m)$ bits |
| false negative | impossible |

$$P_{\text{fp}} \approx \left(1 - e^{-kn/m}\right)^k$$

The false-positive rate rises with the number of inserted elements $n$; for a fixed $m$ there is an optimal $k$.

---

## Source Code

```cpp
void insert(const std::string& key) {
    bits[hash1(key)] = true;
    bits[hash2(key)] = true;
    bits[hash3(key)] = true;
}

bool possiblyContains(const std::string& key) const {
    return bits[hash1(key)] && bits[hash2(key)] && bits[hash3(key)];
}
```

---

## Pros, Cons & When to Use

- Pro: uses far less space than storing the full element set.
- Pro: insertion and query are both constant-time $O(k)$.
- Con: false positives occur, and elements cannot be removed.
- Use for cache pre-filtering, deduplication, and "definitely never seen" checks.

---

## Summary

- A probabilistic set built from a bit array plus several hash functions.
- "Definitely absent" is certain; "possibly present" must tolerate false positives.
- Trades accuracy for a very small space footprint.
