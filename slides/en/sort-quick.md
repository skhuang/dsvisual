## Quick Sort

Quick Sort picks a pivot element, partitions the array into elements less than and greater than the pivot, then recursively sorts both parts. Average $O(n \log n)$ — the most widely used general-purpose sorting algorithm.

---

## Core Concept

`partition` uses the Lomuto scheme with `arr[high]` as pivot; pointer `i` (slow) and `j` (fast). When `arr[j] < pivot`, increment `i` then swap `arr[i]` and `arr[j]`. Finally place the pivot at `arr[i+1]`.

- divide-and-conquer: after partition the pivot is in its final position; left/right sub-problems are independent.
- NOT stable: the swap during partition can reorder equal elements.
- in-place: recursion stack uses $O(\log n)$ average, $O(n)$ worst case.

---

## Operation Flow

1. Choose `pivot = arr[high]`.
2. Set `i = low - 1`; scan `j` from `low` to `high-1`.
3. If `arr[j] < pivot`: increment `i`, then swap `arr[i]` and `arr[j]`.
4. After scanning, swap `arr[i+1]` with `arr[high]` (pivot lands), return `i+1`.

## Complexity Analysis

| Case | Time | Space (stack) |
| --- | --- | --- |
| Best (balanced) | $O(n \log n)$ | $O(\log n)$ |
| Average | $O(n \log n)$ | $O(\log n)$ |
| Worst (sorted/reverse) | $O(n^2)$ | $O(n)$ |
| NOT stable / in-place | — | Aux $O(\log n)$ avg |

$$T_{\text{avg}}(n) = 2T\!\left(\frac{n}{2}\right) + O(n) = O(n \log n)$$

With balanced partitions, recursion depth is $\log n$, each level scans $O(n)$, giving $O(n \log n)$. Worst case (pivot always max/min) degrades to $O(n^2)$.

## Source Code

```cpp
int partition(int arr[], int low, int high) {
    int pivot = arr[high]; // Lomuto: last element as pivot
    int i = low - 1;
    for (int j = low; j < high; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[high]); // place pivot
    return i + 1;
}

void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}
```

## Pros, Cons & When to Use

- Pro: average $O(n \log n)$, cache-friendly — one of the fastest general-purpose sorts in practice.
- Pro: in-place; lower memory overhead than Merge Sort.
- Con: worst case $O(n^2)$ (sorted input + fixed pivot); randomized or median-of-three pivot selection mitigates this.
- Con: NOT stable.
- Use for general array sorting; C++ `std::sort` typically uses Introsort (Quick + Heap hybrid).

---

## Summary

- Pivot-based partition + recursive sort; average $O(n \log n)$, worst $O(n^2)$.
- NOT stable; in-place (stack $O(\log n)$ average).
- Fastest general-purpose sort in practice; randomized pivot makes worst case extremely unlikely.
