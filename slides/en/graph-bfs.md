## Breadth-First Search

BFS (Breadth-First Search) uses a queue to explore graph vertices layer by layer in increasing distance from the source — the standard algorithm for shortest unweighted paths.

---

## Core Concept

A queue tracks nodes to explore next, and a `visited` set prevents revisiting. Each iteration dequeues a node, visits it, then enqueues all unvisited neighbors.

- FIFO queue ensures nodes at the same distance are all visited before exploring farther ones.
- From source 0: distance 0 → {0}, distance 1 → {1, 4}, distance 2 → {2, 3}.
- Can simultaneously maintain a `dist[]` array to record shortest-path distances from source.

---

## Operation Flow

1. Mark source 0 as visited and enqueue it; `visited[0] = true`.
2. Loop: dequeue front node $u$, visit it, mark and enqueue every unvisited neighbor in `adj[u]`.
3. Repeat until the queue is empty; the visit sequence is the BFS order, which equals the level-order traversal.

## Complexity Analysis

| Operation | Time | Space |
| --- | --- | --- |
| Init | $O(V)$ | $O(V)$ |
| BFS traversal | $O(V+E)$ | $O(V)$ |
| Shortest path query | $O(V)$ | $O(V)$ |

$$T_{BFS} = O(V + E)$$

Each vertex and each edge is processed at most once; the queue holds at most $O(V)$ nodes simultaneously.

## Source Code

```cpp
void bfs(const vector<vector<int>>& adj, int start) {
    int n = adj.size();
    vector<bool> visited(n, false);
    queue<int> q;
    q.push(start);
    visited[start] = true;
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        cout << "Visit " << u << "\n";
        for (int v : adj[u]) {
            if (!visited[v]) {
                visited[v] = true;
                q.push(v);
            }
        }
    }
}
```

## Pros, Cons & When to Use

- Pro: naturally finds shortest unweighted paths; guarantees level-order (distance-layer) traversal.
- Pro: computes shortest distances from source to all nodes in a single pass.
- Con: for wide graphs (high branching factor), the queue can hold $O(V)$ nodes, requiring significant memory.
- Use for unweighted shortest paths, level-order traversal, network broadcasting, bipartite checking; pairs naturally with the adjacency list (see graph-adjlist).

---

## Summary

- Queue + visited set; visits all reachable nodes in distance-layer order.
- Time $O(V+E)$, space $O(V)$; finds shortest paths in unweighted graphs.
- Pairs naturally with the adjacency list (see graph-adjlist).
