## Kruskal Minimum Spanning Tree

Kruskal's algorithm greedily picks edges in ascending weight order and uses Disjoint Set Union (DSU / Union-Find) to detect cycles, building a Minimum Spanning Tree (MST) in $O(E \log E)$ time.

---

## Core Concept

An MST spans all vertices with exactly $V-1$ edges and minimum total weight. Kruskal's key insight is using DSU to check connectivity of two vertices in near-$O(1)$ amortized time.

- DSU `find(x)`: with path compression, returns the root of x's set.
- DSU `unite(a,b)`: merges two sets; returns false if already in the same set (cycle detected).
- Edges are sorted by weight and greedily accepted; stop when $V-1$ edges are collected.

---

## Operation Flow

1. Sort all edges by weight in ascending order.
2. Initialise DSU: each vertex is its own set.
3. Take the lightest edge (u,v,w); if u and v are in different sets, add it to MST and merge sets.
4. Repeat until MST has $V-1$ edges.

## Complexity Analysis

| Phase | Time | Space |
| --- | --- | --- |
| Sort edges | $O(E \log E)$ | $O(E)$ |
| DSU ops (E times) | $O(E \alpha(V))$ | $O(V)$ |
| Total Time | $O(E \log E)$ | — |
| Total Space | — | $O(V + E)$ |

$$T_{\text{Kruskal}} = O(E \log E)$$

Sorting dominates the total complexity; DSU operations are amortized $O(\alpha(V)) \approx O(1)$ per operation.

## Source Code

```cpp
struct DSU {
    vector<int> p, r;
    DSU(int n): p(n), r(n, 0) { for (int i = 0; i < n; i++) p[i] = i; }
    int find(int x) { return p[x] == x ? x : p[x] = find(p[x]); }
    bool unite(int a, int b) {
        a = find(a); b = find(b);
        if (a == b) return false;         // cycle
        if (r[a] < r[b]) swap(a, b);
        p[b] = a;
        if (r[a] == r[b]) r[a]++;
        return true;
    }
};

// Kruskal MST
sort(edges.begin(), edges.end(),
     [](const Edge& a, const Edge& b){ return a.w < b.w; });
DSU dsu(V);
vector<Edge> mst;
for (const auto& e : edges) {
    if (dsu.unite(e.u, e.v)) {
        mst.push_back(e);
        if ((int)mst.size() == V - 1) break;
    }
}
```

## Pros, Cons & When to Use

- Pro: edge-centric — well-suited for sparse graphs ($E \ll V^2$).
- Pro: DSU implementation is concise; edges can be sorted offline and processed in bulk.
- Con: requires all edges upfront for sorting — unsuitable for dynamic edge insertions.
- Con: less efficient than Prim's algorithm on dense graphs.
- Use for MST of sparse weighted connected graphs: network cabling, circuit design.

---

## Summary

- Sort edges + DSU cycle detection: MST built in $O(E \log E)$ time.
- MST has exactly $V-1$ edges, spans all vertices with minimum total weight.
- Best for sparse graphs; Prim's algorithm is preferred for dense graphs.
