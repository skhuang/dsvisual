## Insertion Sort

Insertion Sort mimics sorting a hand of cards: pick up the next card (element) one at a time and insert it into the correct position within the already-sorted left portion, until all cards are ordered. Nearly-sorted data achieves $O(n)$.

---

## Core Concept

The outer loop starts at `i=1` and extracts `key = arr[i]`; the inner loop shifts all sorted elements greater than `key` one position right, then inserts `key` into the vacated slot.

- The sorted prefix `arr[0..i-1]` is always maintained in order.
- stable: equal elements are never swapped — original order is preserved.
- in-place: only $O(1)$ extra space.

---

## Operation Flow

1. Take `key = arr[i]`, set `j = i - 1`.
2. While `j >= 0` and `arr[j] > key`: set `arr[j+1] = arr[j]`, `j--` (shift right).
3. Set `arr[j+1] = key` (insert at the correct position).
4. Increment `i` and process the next element.

## Complexity Analysis

| Case | Time | Space |
| --- | --- | --- |
| Best (sorted) | $O(n)$ | $O(1)$ |
| Average | $O(n^2)$ | $O(1)$ |
| Worst (reverse) | $O(n^2)$ | $O(1)$ |
| stable / in-place | Yes | Auxiliary $O(1)$ |

$$T_{\text{best}}(n) = O(n),\quad T_{\text{worst}}(n) = O(n^2)$$

Best case: sorted input exits the inner loop immediately each pass — only $n-1$ comparisons total; worst case is reverse order, requiring $i$ shifts in pass $i$.

## Source Code

```cpp
void insertionSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        // Shift elements greater than key one position right
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j = j - 1;
        }
        arr[j + 1] = key; // insert
    }
}
```

## Pros, Cons & When to Use

- Pro: stable, in-place; $O(n)$ for nearly-sorted data.
- Pro: lower constant overhead than Quick/Merge Sort for small datasets.
- Pro: online algorithm — can insert elements one by one as they arrive.
- Con: average and worst case $O(n^2)$; not suitable for large datasets.
- Use for small arrays, nearly-sorted streaming data, or as the gap=1 final phase of Shell Sort.

---

## Summary

- Inserts each element into the sorted prefix one at a time; stable, in-place.
- Best $O(n)$, average/worst $O(n^2)$.
- One of the best choices for small or nearly-sorted datasets.
