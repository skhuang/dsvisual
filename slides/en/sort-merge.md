## Merge Sort

Merge Sort uses divide-and-conquer: recursively split the array into halves until single elements, then merge sorted sub-arrays back up. Guarantees $O(n \log n)$ in all cases and is stable.

---

## Core Concept

`mergeSort(arr, l, r)` finds midpoint `m = l + (r-l)/2`, recursively sorts `[l,m]` and `[m+1,r]`, then calls `merge`. `merge` creates temporary arrays L and R, then uses two pointers to write the smaller element back one by one.

- During merge, taking L[i] when L[i] <= R[j] ensures equal elements keep their relative order (stable).
- NOT in-place: merge requires $O(n)$ auxiliary space.
- Recursion depth is $O(\log n)$; each level merges $O(n)$ total, giving $O(n \log n)$ overall.

---

## Operation Flow

1. If `l >= r`, return (base case: single element).
2. Compute `m = l + (r-l)/2`, recurse `mergeSort(arr, l, m)` and `mergeSort(arr, m+1, r)`.
3. Call `merge(arr, l, m, r)`: copy into L and R, use two pointers to merge back into `arr[l..r]`.
4. Copy any remaining elements of L or R back.

## Complexity Analysis

| Case | Time | Space |
| --- | --- | --- |
| Best | $O(n \log n)$ | $O(n)$ |
| Average | $O(n \log n)$ | $O(n)$ |
| Worst | $O(n \log n)$ | $O(n)$ |
| stable / NOT in-place | — | Auxiliary $O(n)$ |

$$T(n) = 2T\!\left(\frac{n}{2}\right) + O(n) = O(n \log n)$$

Recursion tree has depth $\log n$; each level merges $O(n)$; by master theorem $T(n) = O(n \log n)$ in all cases.

## Source Code

```cpp
void merge(int arr[], int l, int m, int r) {
    int n1 = m - l + 1, n2 = r - m;
    int L[n1], R[n2];
    for (int i = 0; i < n1; i++)
        L[i] = arr[l + i];
    for (int j = 0; j < n2; j++)
        R[j] = arr[m + 1 + j];
    int i = 0, j = 0, k = l;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j])
            arr[k++] = L[i++];
        else
            arr[k++] = R[j++];
    }
    while (i < n1)
        arr[k++] = L[i++];
    while (j < n2)
        arr[k++] = R[j++];
}

void mergeSort(int arr[], int l, int r) {
    if (l >= r)
        return;
    int m = l + (r - l) / 2;
    mergeSort(arr, l, m);
    mergeSort(arr, m + 1, r);
    merge(arr, l, m, r);
}
```

## Pros, Cons & When to Use

- Pro: stable, $O(n \log n)$ in all cases — no worst-case degradation.
- Pro: ideal for linked-list sorting (merge does not require random access).
- Pro: suitable for external sort: can read and merge disk blocks in batches.
- Con: requires $O(n)$ auxiliary space — more memory overhead than Quick Sort.
- Use when stability is required, for linked-list sorting, external sort, or when worst-case guarantees are critical.

---

## Summary

- divide-and-conquer: recursive split + linear merge.
- stable; $O(n \log n)$ always; $O(n)$ auxiliary space.
- The go-to algorithm for external sorting and linked-list sorting.
