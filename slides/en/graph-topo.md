## Topological Sort (Kahn's Algorithm)

Topological Sort arranges the vertices of a Directed Acyclic Graph (DAG) so that for every directed edge u→v, u appears before v. Kahn's BFS-based algorithm repeatedly removes vertices with in-degree 0 in $O(V+E)$ time.

---

## Core Concept

In-degree is the count of edges pointing into a vertex. A vertex with in-degree 0 has no prerequisites and can safely be placed first. Removing it decrements neighbors' in-degrees, potentially unlocking new zero-in-degree vertices.

- DAG requirement: topological sort is only valid for acyclic graphs; if fewer than V vertices are processed, a cycle exists.
- Non-uniqueness: if multiple vertices have in-degree 0 simultaneously, the ordering is not unique.
- Applications: task scheduling, dependency resolution, instruction ordering in compilers.

---

## Operation Flow

1. Compute in-degrees of all vertices; enqueue all vertices with in-degree 0.
2. Dequeue vertex u and append it to the topological order.
3. For each successor v of u, decrement `inDegree[v]`; if `inDegree[v] == 0` enqueue v.
4. Repeat until the queue is empty; if result length equals V the sort succeeded, else a cycle exists.

## Complexity Analysis

| Phase | Time | Space |
| --- | --- | --- |
| In-degree computation | $O(V+E)$ | $O(V)$ |
| BFS processing | $O(V+E)$ | $O(V)$ |
| Total Space | — | $O(V+E)$ |

$$T_{\text{Kahn}} = O(V + E)$$

Each vertex is enqueued and dequeued exactly once; each edge is processed once — linear time is optimal for topological sort.

## Source Code

```cpp
// Kahn's Algorithm (BFS-based topological sort)
vector<int> inDegree(V, 0);
for (auto [u, v] : edges) {
    adj[u].push_back(v);
    inDegree[v]++;
}

queue<int> q;
for (int i = 0; i < V; i++)
    if (inDegree[i] == 0) q.push(i);

vector<int> topoOrder;
while (!q.empty()) {
    int u = q.front(); q.pop();
    topoOrder.push_back(u);
    for (int v : adj[u]) {
        if (--inDegree[v] == 0)
            q.push(v);
    }
}

if ((int)topoOrder.size() != V)
    cout << "ERROR: Cycle detected!\n";
```

## Pros, Cons & When to Use

- Pro: linear $O(V+E)$ time; cycle detection comes for free.
- Pro: Kahn's BFS approach is intuitive and easy to debug.
- Con: only valid for DAGs; directed graphs with cycles produce no valid topological order.
- Con: when multiple valid orderings exist, the result depends on queue order and is not unique.
- Use for dependency resolution, Makefile build order, course prerequisite scheduling, etc.

---

## Summary

- Repeatedly removing in-degree-0 vertices achieves topological sort in $O(V+E)$ linear time.
- Only for DAGs; if result length is less than V a cycle exists — doubles as a cycle-detection tool.
- Widely applied in task scheduling, dependency resolution, and compiler front-end instruction ordering.
