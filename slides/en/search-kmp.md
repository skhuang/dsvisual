## KMP String Matching

KMP (Knuth-Morris-Pratt) avoids re-comparing characters by precomputing how far the pattern can shift on a mismatch using the LPS array; total time is $O(n+m)$.

---

## Core Concept

The **LPS array** (longest proper prefix that is also a suffix, a.k.a. the failure function) records, at every pattern index, the length of the longest proper prefix which is also a suffix. On a mismatch at index `j`, jump to `lps[j-1]` instead of restarting.

- The LPS array is built in $O(m)$ time.
- The text index `i` never moves backward; it advances by at least one step each time.
- Total comparisons $\le 2n$; overall complexity $O(n+m)$.

---

## Operation Flow

1. **Preprocess**: run `computeLPS` on the pattern to build the LPS array.
2. **Match advance**: when `text[i] == pat[j]` increment both `i` and `j`; when `j == m` record a match and set `j = lps[j-1]`.
3. **Mismatch**: if `j > 0`, set `j = lps[j-1]` (keep `i` fixed); if `j == 0`, increment `i`.

## Complexity Analysis

| Phase | Time | Space |
| --- | --- | --- |
| Build LPS | $O(m)$ | $O(m)$ |
| Search | $O(n)$ | $O(1)$ |
| Total | $O(n+m)$ | $O(m)$ |

$$T_{\text{KMP}} = O(n + m)$$

KMP is linear-time: the text pointer never retreats, so no character is scanned more than twice.

## Source Code

```cpp
vector<int> computeLPS(const string& pat) {
    int m = pat.size();
    vector<int> lps(m, 0);
    int len = 0;
    for (int i = 1; i < m;) {
        if (pat[i] == pat[len]) {
            lps[i++] = ++len;
        } else if (len != 0) {
            len = lps[len - 1];
        } else {
            lps[i++] = 0;
        }
    }
    return lps;
}

void kmpSearch(const string& text, const string& pat) {
    int n = text.size(), m = pat.size();
    vector<int> lps = computeLPS(pat);
    int i = 0, j = 0;
    while (i < n) {
        if (text[i] == pat[j]) {
            i++;
            j++;
            if (j == m) {
                cout << "Match at index " << (i - j) << endl;
                j = lps[j - 1];
            }
        } else if (j != 0) {
            j = lps[j - 1];
        } else {
            i++;
        }
    }
}
```

## Pros, Cons & When to Use

- Pro: guaranteed linear time; the text pointer never backtracks.
- Pro: the LPS array is straightforward to precompute in $O(m)$ time and space.
- Con: constant factor higher than Boyer-Moore; slower in practice on typical text.
- Con: does not exploit the alphabet; cannot make large skips like Boyer-Moore.
- Use when worst-case linearity matters, e.g. highly repetitive strings.

---

## Summary

- The LPS (failure function) array is the key precomputed data structure in KMP.
- The text pointer never retreats; total comparisons $\le 2n$.
- Time $O(n+m)$, space $O(m)$.
- For a comparison with BM and RK, see search-strcompare.
