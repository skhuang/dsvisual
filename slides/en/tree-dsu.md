## Disjoint Set (Union-Find)

A Disjoint Set (Union-Find) represents multiple disjoint sets as a forest and supports near-$O(1)$ union and find operations — the core data structure for Kruskal's MST and connectivity queries.

---

## Core Concept

Each set is a tree where `parent[i]` points to the parent; the root satisfies `parent[root] == root`. `find` follows parent pointers to the root; `union` attaches one root under the other.

- Path compression: during `find`, redirect every visited node straight to the root, flattening future lookups.
- Union by rank: attach the root with smaller rank under the root with larger rank to keep tree height low.
- Together they achieve amortized $O(\alpha(N))$ time — effectively $O(1)$ for all practical $N$.

---

## Operation Flow

1. Init: each element is its own set; `parent[i] = i`, `rank[i] = 0`.
2. `find(x)`: if `parent[x] == x` return `x`; otherwise recurse `find(parent[x])` and path-compress.
3. `union(a, b)`: find roots of both sets; if different, attach smaller-rank root under larger-rank root (if equal, pick one and increment its rank).

## Complexity Analysis

| Operation | Time | Space |
| --- | --- | --- |
| Init | $O(N)$ | $O(N)$ |
| find (with path compression) | $O(\alpha(N))$ | $O(1)$ |
| unite (with union by rank) | $O(\alpha(N))$ | $O(1)$ |

$$O(\alpha(N)) \approx O(1) \text{ for all practical } N$$

$\alpha$ is the inverse Ackermann function; it is $\leq 4$ for all practical $N$, making operations effectively constant time.

## Source Code

```cpp
struct DSU {
    vector<int> p, r;
    DSU(int n) : p(n), r(n, 0) {
        for (int i = 0; i < n; i++)
            p[i] = i;
    }
    int find(int x) {
        return p[x] == x ? x : p[x] = find(p[x]); // path compression
    }
    bool unite(int a, int b) {
        a = find(a); b = find(b);
        if (a == b) return false;
        if (r[a] < r[b]) swap(a, b);
        p[b] = a; // union by rank
        if (r[a] == r[b]) r[a]++;
        return true;
    }
};
```

## Pros, Cons & When to Use

- Pro: simple to implement; amortized near-$O(1)$; extremely fast in practice.
- Pro: widely applicable — Kruskal's MST, network connectivity, image segmentation.
- Con: does not support efficient split or un-union operations.
- Use when you need fast same-set queries and set-merge operations.

---

## Summary

- parent[] array + path compression + union by rank → amortized $O(\alpha(N))$ per operation.
- For all practical sizes, $\alpha(N) \leq 4$, making each operation effectively constant time.
- The workhorse for Kruskal's MST (see graph-kruskal).
