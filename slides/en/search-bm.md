## Boyer-Moore String Matching

Boyer-Moore scans the pattern right-to-left and uses two heuristics — bad-character and good-suffix — to skip large chunks of text, often achieving sublinear performance in practice.

---

## Core Concept

The **bad-character** heuristic aligns the mismatched text character with its rightmost occurrence in the pattern. The **good-suffix** heuristic reuses the already-matched suffix to find the next alignment. The shift applied is the `max` of both.

- Right-to-left scanning enables large jumps on every mismatch.
- Preprocessing time is $O(m + \sigma)$ where $\sigma$ is the alphabet size.
- Best case skips $m$ characters at a time, giving $O(n/m)$ total comparisons.

---

## Operation Flow

1. **Preprocess**: call `buildBadChar` to build the bad-character table and `buildGoodSuffix` to build the good-suffix shift table.
2. **Right-to-left compare**: match from `j = m-1` downward; stop on the first mismatch.
3. **Shift**: on a full match apply `shift[0]`; on mismatch shift by `max(good-suffix, bad-character)`.

## Complexity Analysis

| Phase | Time | Space |
| --- | --- | --- |
| Preprocessing | $O(m + \sigma)$ | $O(m + \sigma)$ |
| Search (best) | $O(n/m)$ | — |
| Search (worst) | $O(nm)$ | — |

$$T_{\text{BM,best}} = O(n/m)$$

On typical text BM is sublinear; the worst case (highly repetitive strings) degrades to $O(nm)$, avoidable with the Galil rule.

## Source Code

```cpp
vector<int> buildBadChar(const string& pat) {
    vector<int> badChar(ALPHABET, -1);
    for (int i = 0; i < (int)pat.size(); i++)
        badChar[(unsigned char)pat[i]] = i;
    return badChar;
}

void buildGoodSuffix(const string& pat, vector<int>& shift) {
    int m = pat.size();
    vector<int> bpos(m + 1, 0);
    shift.assign(m + 1, 0);
    int i = m, j = m + 1;
    bpos[i] = j;
    while (i > 0) {
        while (j <= m && pat[i - 1] != pat[j - 1]) {
            if (shift[j] == 0)
                shift[j] = j - i;
            j = bpos[j];
        }
        i--;
        j--;
        bpos[i] = j;
    }
    j = bpos[0];
    for (i = 0; i <= m; i++) {
        if (shift[i] == 0)
            shift[i] = j;
        if (i == j)
            j = bpos[j];
    }
}

void boyerMooreSearch(const string& text, const string& pat) {
    int n = text.size(), m = pat.size();
    vector<int> badChar = buildBadChar(pat);
    vector<int> shift;
    buildGoodSuffix(pat, shift);
    int s = 0;
    while (s <= n - m) {
        int j = m - 1;
        while (j >= 0 && pat[j] == text[s + j])
            j--;
        if (j < 0) {
            cout << "Match at index " << s << endl;
            s += shift[0];
        } else {
            int bcShift = j - badChar[(unsigned char)text[s + j]];
            s += max(shift[j + 1], max(1, bcShift));
        }
    }
}
```

## Pros, Cons & When to Use

- Pro: sublinear in practice; the fastest general-purpose matcher on large alphabets and long texts.
- Pro: right-to-left scanning often yields much larger skips than KMP.
- Con: worst case is $O(nm)$ without the Galil rule.
- Con: two preprocessing tables (bad-character and good-suffix) make the implementation more complex.
- Use for large alphabets (e.g. natural language text) and long patterns.

---

## Summary

- Right-to-left scan; bad-character and good-suffix heuristics.
- Average sublinear; worst case $O(nm)$.
- The go-to algorithm for large alphabets and long texts.
- For a comparison with KMP and RK, see search-strcompare.
