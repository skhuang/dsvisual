## Bellman-Ford

Bellman-Ford computes single-source shortest paths and, unlike Dijkstra, works with negative edge weights; it can also detect negative cycles.

---

## Core Concept

Relaxing an edge `(u, v)` means `dist[v] = min(dist[v], dist[u] + w)`.

- One pass relaxes every edge once.
- After `V-1` passes all shortest paths have converged (a shortest path has at most `V-1` edges).
- A `V`-th pass that still relaxes something proves a negative cycle exists.

---

## Operation Flow

1. Set `dist[source] = 0` and all others to infinity.
2. Repeat `V-1` times — relax every edge.
3. Optionally run one more pass to check for a negative cycle.

## Complexity Analysis

| Item | Result |
| --- | --- |
| time | $O(V \cdot E)$ |
| space | $O(V)$ |
| handles negative weights | yes |

$$T(V, E) = O(V \cdot E)$$

`V-1` passes, each relaxing all `E` edges.

---

## Source Code

```cpp
std::vector<int> dist(V, INT_MAX);
dist[0] = 0;

for (int pass = 1; pass <= V - 1; pass++) {
    for (const Edge& e : edges) {
        if (dist[e.u] != INT_MAX && dist[e.u] + e.w < dist[e.v])
            dist[e.v] = dist[e.u] + e.w;
    }
}
```

---

## Pros, Cons & When to Use

- Pro: handles negative weights, detects negative cycles, simple to code.
- Con: slower than Dijkstra.
- Use: graphs with negative edges, or when negative-cycle detection is required.

---

## Summary

- Relax every edge for `V-1` passes.
- Converges because a shortest path has at most `V-1` edges.
- A further relaxation signals a negative cycle.
