## Publish-Subscribe Pattern

Publishers emit events to an event bus (broker); subscribers receive them from the bus — publishers and subscribers never reference each other.

---

## Core Concept

The event bus is the key to decoupling: it hides each side from the other. One event fans out to many subscribers; subscribers can be added or removed at runtime.

- Publishers and subscribers are fully decoupled — neither depends on the other.
- A single event can be broadcast to multiple subscribers simultaneously (fan-out).
- Subscribers can be dynamically added or removed at runtime for maximum flexibility.

---

## Operation Flow

1. Subscribers register their handler functions with the event bus.
2. A publisher calls the bus's publish() method with the event data.
3. The bus invokes every registered handler in sequence, completing the event dispatch.

## Trade-offs & When to Use

| Aspect | Notes |
| --- | --- |
| Decoupling | Publishers and subscribers have zero direct dependency on each other |
| Fan-out | One event can easily reach multiple subscribers simultaneously |
| Cost | Event flow is harder to trace; ordering and delivery guarantees need explicit attention |

## Source Code

```cpp
// Event bus — decouples publishers from subscribers.
class EventBus {
    vector<function<void(const string&)>> subscribers;

public:
    void subscribe(function<void(const string&)> handler) {
        subscribers.push_back(handler);
    }
    void publish(const string& event) {
        for (auto& handler : subscribers)
            handler(event);
    }
};
```

---

## Pros, Cons & When to Use

- Pro: publishers and subscribers are totally decoupled — neither knows of the other.
- Pro: adding or removing subscribers requires no changes to the publisher.
- Pro: naturally supports one-to-many event fan-out.
- Con: indirect control flow makes debugging and tracing harder.
- Use for event-driven systems, UI event handling, and decoupled inter-module communication.

---

## Summary

- Publish-Subscribe uses an event bus as a broker to fully decouple publishers and subscribers.
- One event can broadcast to any number of subscribers, with dynamic subscription at runtime.
- It is the core mechanism of event-driven architecture, widely used in UI frameworks and messaging systems.
