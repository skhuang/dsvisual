## Dijkstra Shortest Path

Dijkstra's algorithm greedily selects the unvisited vertex with minimum distance, then relaxes its outgoing edges. With a binary heap it runs in $O((V+E)\log V)$ and finds shortest paths from a source to all vertices.

---

## Core Concept

Maintain a distance array `dist[]` initialised to INF except the source (0). A min-heap priority queue ensures the next vertex popped has the smallest known distance; once visited, a vertex's distance is final.

- relax: if `dist[u] + w(u,v) < dist[v]`, update `dist[v]` and push (dist[v], v) into the priority queue.
- Lazy deletion: if a popped vertex is already visited, skip it.
- Limitation: edge weights must be non-negative; use Bellman-Ford for negative weights.

---

## Operation Flow

1. Initialise `dist[source]=0`, all others INF; push source into the priority queue.
2. Pop vertex u with minimum distance; skip if already visited.
3. Mark u as visited; relax each edge (u,v,w) of u.
4. Repeat until the priority queue is empty; `dist[]` then holds all shortest distances.

## Complexity Analysis

| Phase | Time | Space |
| --- | --- | --- |
| Initialisation | $O(V)$ | $O(V)$ |
| PQ ops (V pops) | $O(V \log V)$ | $O(V)$ |
| relax (E pushes) | $O(E \log V)$ | $O(E)$ |
| Total Space | — | $O(V + E)$ |

$$T_{\text{Dijkstra}} = O((V+E)\log V)$$

With a binary heap; using a Fibonacci heap reduces this to $O(E + V \log V)$ at the cost of implementation complexity.

## Source Code

```cpp
const int INF = 1e9;
vector<int> dist(V, INF);
vector<bool> visited(V, false);
priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq;

dist[source] = 0;
pq.push({0, source});

while (!pq.empty()) {
    auto [d, u] = pq.top();
    pq.pop();
    if (visited[u])
        continue;
    visited[u] = true;

    for (auto [v, w] : adj[u]) {
        if (!visited[v] && dist[u] + w < dist[v]) {
            dist[v] = dist[u] + w;
            pq.push({dist[v], v}); // relax
        }
    }
}
```

## Pros, Cons & When to Use

- Pro: guarantees shortest paths on graphs with non-negative edge weights; binary heap gives good practical performance.
- Pro: one run produces shortest distances from the source to all other vertices.
- Con: cannot handle negative edges; Bellman-Ford ($O(VE)$) or SPFA is needed.
- Con: lazy deletion may accumulate $O(E)$ stale entries in the priority queue.
- Use for map navigation, network routing (OSPF), game AI pathfinding, etc.

---

## Summary

- Min-heap + relax: expand the vertex with minimum distance each time until all distances are finalised.
- Binary heap complexity $O((V+E)\log V)$; valid only for graphs with non-negative edge weights.
- Widely used in map navigation and network routing for single-source shortest path problems.
