## BFS vs DFS Compared

On the same 5-node graph (0:[1,4], 1:[0,2,3,4], 2:[1,3], 3:[1,2,4], 4:[0,1,3]), BFS from 0 visits 0 1 4 2 3 and DFS from 0 visits 0 1 2 3 4; the visualizer steps both panes in sync to make the contrast visible.

---

## Core Concept

BFS uses a Queue and explores layer by layer from nearest to farthest, naturally finding shortest unweighted paths; DFS uses a Stack or recursion, diving deep before backtracking.

- BFS: queue; visit order 0→1→4→2→3; layers d=0,1,1,2,2.
- DFS: stack/recursion; visit order 0→1→2→3→4; explores paths fully before backtracking.
- Both have time $O(V+E)$; space differs: BFS $O(V)$ queue vs DFS $O(h)$ stack depth.

---

## Operation Flow

1. BFS: enqueue source 0 → dequeue and visit → enqueue unvisited neighbors in order → repeat until empty.
2. DFS: push source 0 → pop; if unvisited mark it and push neighbors in reverse order → repeat.
3. Comparison: same graph, same source, but different control structure → different visit orders.

## Complexity Analysis

| Operation | Time | Space |
| --- | --- | --- |
| Init | $O(V)$ | $O(V)$ |
| BFS traversal | $O(V+E)$ | $O(V)$ |
| DFS traversal | $O(V+E)$ | $O(h)$ |
| Single step $\deg(v)$ | $O(\deg(v))$ | $O(1)$ |

$$T_{BFS} = T_{DFS} = O(V+E)$$

Both have the same time complexity; the key difference is space: BFS queue holds up to $O(V)$ nodes, while DFS stack depth is at most $O(h)$ (DFS tree height).

## Source Code

```cpp
vector<int> bfsOrder(const vector<vector<int>>& adj, int start) {
    int n = adj.size();
    vector<bool> visited(n, false);
    vector<int> order;
    queue<int> q;
    q.push(start); visited[start] = true;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : adj[u])
            if (!visited[v]) { visited[v] = true; q.push(v); }
    }
    return order;
}
vector<int> dfsOrder(const vector<vector<int>>& adj, int start) {
    int n = adj.size();
    vector<bool> visited(n, false);
    vector<int> order;
    stack<int> s;
    s.push(start);
    while (!s.empty()) {
        int u = s.top(); s.pop();
        if (visited[u]) continue;
        visited[u] = true; order.push_back(u);
        for (auto it = adj[u].rbegin(); it != adj[u].rend(); ++it)
            if (!visited[*it]) s.push(*it);
    }
    return order;
}
```

## Pros, Cons & When to Use

- BFS pro: finds shortest unweighted path; good for level-order traversal and broadcast problems.
- DFS pro: uses less memory for deep narrow graphs; foundation for topological sort, SCC, cycle detection.
- BFS con: high memory on wide graphs (queue can reach $O(V)$).
- DFS con: does not guarantee shortest path; different neighbor orderings yield different results, which can be confusing.

---

## Summary

- Same data (graph), different control structure (queue vs stack) → completely different traversal orders.
- BFS expands by distance layers; DFS explores paths fully before backtracking — each suits different problems.
- Both run in $O(V+E)$ time; BFS uses $O(V)$ space, DFS uses $O(h)$ stack space.
