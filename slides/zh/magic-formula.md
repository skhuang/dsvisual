## 從陣列到公式:一個計算欄位

前面兩節都先建出整個 $n \times n$ 方陣,再從裡面讀值。這一節換個角度:能不能完全不建方陣,單靠 $(i,j)$ 直接算出那一格的值?

> 這正是「計算欄位(computed property)取代已儲存表格」的經典取捨——就像用完美雜湊(perfect hash)取代查表,或用 getter 函式取代快取欄位。

---

## 兩個數字平面其實是線性的

上一節看到 $a = \lfloor (v-1)/n \rfloor$、$b = (v-1) \bmod n$ 兩個平面各自是拉丁方陣。把 Siamese/Coxeter 的填數規則展開,可以證明 $a$、$b$ 其實是 $(i,j)$ 的線性函數(取 mod $n$)。

$$a = \left(i - j + \frac{n-1}{2}\right) \bmod n, \qquad b = \big(i - 2j + (n-1)\big) \bmod n$$

兩個係數 $(n-1)/2$、$n-1$ 只跟 $n$ 有關,對每一格都相同。

> C++ 的 <code>%</code> 對負數可能傳回負值,所以要用 <code>((x % m) + m) % m</code> 保護一下,確保 mod 結果落在 $0..n-1$。

---

## 重組回值:O(1) 公式

1. 算 $a = (i-j+(n-1)/2) \bmod n$。
2. 算 $b = (i-2j+(n-1)) \bmod n$。
3. 套用分解恆等式 $value = n \cdot a + b + 1$。
4. 全程沒有讀寫任何 $n \times n$ 陣列。

```cpp
int getValue(int n, int i, int j) {
    int a = mod(i - j + (n - 1) / 2, n);
    int b = mod(i - 2 * j + (n - 1), n);
    return n * a + b + 1;
}
```

---

## 公式 vs. 已儲存的表格

| 作法 | 查一格 | 額外空間 | 填滿全部 |
| --- | --- | --- | --- |
| 循序建表(Siamese) | 需先建完整表,$O(n^2)$ | $O(n^2)$(整個方陣 + 已訪記錄) | $O(n^2)$ |
| 本節公式 | $O(1)$,任意順序、可重複查詢 | $O(1)$ | $O(n^2)$(逐格各自 $O(1)$) |

> 這對應「空間複雜度優化」的常見手法:當一個值可以由少數參數即時算出時,不需要把它整個快取/儲存下來。

---

## 小結:與拉丁分解的關係

- 這裡的 $a$、$b$ 正是 <code>magic-latin</code> 那兩個拉丁方陣平面,只是這次直接用 $(i,j)$ 算,不必先建出方陣再讀出來。
- 因為 $a$、$b$ 是線性且各自是拉丁方陣,每列/欄/對角線總和固定的性質依然成立。
- 「已建表可查」與「隨算隨查、不建表」是同一個問題的兩種答案:選哪一種,取決於你要重複查很多次(建表划算)還是只查少數幾格(公式划算)。
