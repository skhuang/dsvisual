## D-ary 堆積(4-ary Heap)

Binary Heap 的推廣:每個節點有 $d$ 個子節點($d=4$ 為 4-ary Heap),以更寬的樹減少 sift-up 次數,但 sift-down 每層需比較 $d$ 個子節點。

---

## 核心概念

節點 $i$ 的父節點為 $(i-1)/d$;第 $j$ 個子節點為 $i \cdot d + j + 1$($j = 0, \ldots, d-1$)。高度為 $\lfloor\log_d N\rfloor$,比 Binary Heap 矮。

- sift-up:每層只有一次比較(與父節點),步數為 $O(\log_d N)$,比 Binary Heap 少。
- sift-down:每層需比較 $d$ 個子節點;總代價 $O(d \log_d N)$。
- insert 優化:大量 insert 場景下 $d > 2$ 可顯著減少 sift-up 的層數。

---

## 運作流程

1. insert(x):追加至陣列末端,執行 sift-up。每步比較 data[i] 與父節點 data[(i-1)/d]。
2. extractTop:根與末端互換後移除末端。
3. sift-down:從根開始,在 $d$ 個子節點中找最小者(min-heap);若小於當前節點則互換並繼續。
4. 重複直到葉節點或不再違反 heap 性質。

> 4-ary Heap:根節點(i=0)有 4 個子節點(i=1..4),每個子節點再各有 4 個子節點(i=5..20...)。高度為 $\lfloor\log_4 N\rfloor$。

## 複雜度分析

| 操作 | 時間(最壞) | 空間 |
| --- | --- | --- |
| insert | $O(\log_d N)$ | $O(1)$ |
| peek | $O(1)$ | $O(1)$ |
| extractTop | $O(d \cdot \log_d N)$ | $O(1)$ |
| decreaseKey | $O(\log_d N)$ | $O(1)$ |
| 空間合計 | — | $O(N)$ |

$$T_{\text{siftDown}}(N) = O(d \cdot \log_d N)$$

sift-down 每層比較 $d$ 個子節點,樹高 $\log_d N$,故總比較次數為 $d \cdot \log_d N$。對 $d=4$:每層 4 次比較,高度 $\log_4 N \approx 0.5 \log_2 N$。

## 程式碼

```cpp
void siftUp(int index) {
    while (index > 0) {
        int p = (index - 1) / d;
        if (!cmp(data[index], data[p])) break;
        swap(data[index], data[p]);
        index = p;
    }
}

void siftDown(int index) {
    while (true) {
        int best = index;
        for (int offset = 0; offset < d; ++offset) {
            int c = index * d + offset + 1;
            if (c < (int)data.size() && cmp(data[c], data[best]))
                best = c;
        }
        if (best == index) break;
        swap(data[index], data[best]);
        index = best;
    }
}

void insert(int value) {
    data.push_back(value);
    siftUp((int)data.size() - 1);
}

int extractTop() {
    int top = data[0];
    data[0] = data.back(); data.pop_back();
    if (!data.empty()) siftDown(0);
    return top;
}
```

## 優缺點與使用時機

- 優點:sift-up 步數為 $O(\log_d N)$,大 $d$ 時顯著少於 Binary Heap。
- 優點:陣列連續儲存,快取友好。
- 缺點:sift-down 每層比較 $d$ 個子節點,$d$ 大時單次 extract 比較次數多。
- 缺點:最佳 $d$ 取決於工作負載(insert vs extract 比例)及快取大小,需調參。
- 適用:insert 遠多於 extract 的場景(如 Dijkstra 在稀疏圖);$d=4$ 通常是實踐中的良好預設。

---

## 小結

- 每節點有 $d$ 個子節點;父節點 $(i-1)/d$,第 $j$ 子 $id+j+1$。
- insert $O(\log_d N)$;extractTop $O(d\log_d N)$;peek $O(1)$。
- $d=4$ 在 insert 密集的工作負載下通常優於 Binary Heap;$d=2$ 在 extract 密集時更佳。
