## 鄰接串列

以每個頂點為起點的鏈結串列來表示圖的邊；對稀疏圖比鄰接矩陣更省記憶體。

---

## 核心概念

一個大小為 $V$ 的陣列，每格 `adj[v]` 是一條串列，儲存 $v$ 的所有鄰居。無向圖中每條邊 $(u, v)$ 同時加進 `adj[u]` 與 `adj[v]`。

- 空間 $O(V + E)$；對稀疏圖明顯優於矩陣的 $O(V^2)$。
- 遍歷某頂點的鄰居 $O(\deg(v))$，與該頂點度數成正比。
- 判斷某條邊存在需 $O(\deg(v))$；矩陣為 $O(1)$。

---

## 運作流程

1. 建立大小為 $V$ 的串列陣列，每格初始為空。
2. 插入邊 $(u, v)$ 時，把 $v$ 追加到 `adj[u]`，把 $u$ 追加到 `adj[v]`（無向圖）。
3. 遍歷頂點 $u$ 的所有鄰居時，逐項走 `adj[u]` 串列。

## 複雜度分析

| 操作 | 時間 | 空間 |
| --- | --- | --- |
| 建構 | $O(V + E)$ | $O(V + E)$ |
| 插入邊 | $O(1)$ | $O(1)$ |
| 遍歷鄰居 | $O(\deg(v))$ | $O(1)$ |
| 邊查詢 | $O(\deg(v))$ | $O(1)$ |
| 空間合計 | — | $O(V + E)$ |

$$O(V + E) \;\ll\; O(V^2) \text{ when } E \ll V^2$$

稀疏圖（$E \ll V^2$）時鄰接串列的空間明顯較佳。

## 程式碼

```cpp
class Graph {
    int V;
    vector<list<int>> adj;
public:
    Graph(int v) : V(v), adj(v) {}
    void addEdge(int u, int v) {
        adj[u].push_back(v);
        adj[v].push_back(u);
    }
};
```

---

## 優缺點與使用時機

- 優點：稀疏圖空間 $O(V+E)$ 比矩陣 $O(V^2)$ 小很多。
- 優點：遍歷鄰居只走 $\deg(v)$ 個元素，不需掃整列。
- 缺點：邊查詢需走串列（$O(\deg(v))$），矩陣 $O(1)$。
- 適用：稀疏圖、邊集合操作為主、需要遍歷鄰居的演算法（BFS / DFS / Dijkstra）。

---

## 小結

- 陣列 + 串列；每邊存兩次（無向）。
- 空間 $O(V+E)$，遍歷鄰居 $O(\deg(v))$。
- 對稀疏圖是首選；BFS / DFS / Dijkstra / Kruskal 內部皆使用此表示。
