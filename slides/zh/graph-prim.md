## Prim 最小生成樹

Prim 演算法以成長單一棵樹的方式建構最小生成樹，每一步都加入離開當前樹的最便宜邊。

---

## 核心概念

維護一組樹內頂點，並為每個樹外頂點記錄連往樹的最便宜已知邊。

- 每一步挑選擁有最小該邊的樹外頂點並把它納入。
- 切割性質保證所選的邊是安全的。
- 與 Kruskal 形成對比，後者對所有邊做全域排序。

---

## 運作流程

1. 從任一頂點開始建樹。
2. 在所有橫跨樹/非樹切割的邊中，挑出權重最小的一條。
3. 把它的端點加入樹。
4. 重複直到所有頂點都進入樹。

## 複雜度分析

| 實作方式 | 複雜度 |
| --- | --- |
| 二元堆 | $O(E \log V)$ |
| 鄰接矩陣掃描 | $O(V^2)$ |
| 空間 | $O(V + E)$ |

$$T(V, E) = O(E \log V)$$

二元堆讓 Prim 在稀疏圖上也能高效運作。

## 程式碼

```cpp
for (int count = 0; count < V; count++) {
    int u = -1;
    for (int v = 0; v < V; v++)
        if (!inMST[v] && (u == -1 || key[v] < key[u]))
            u = v;
    inMST[u] = true;
    for (int v = 0; v < V; v++)
        if (w[u][v] && !inMST[v] && w[u][v] < key[v]) {
            key[v] = w[u][v];
            parent[v] = u;
        }
}
```

## 優缺點與使用時機

- 優點：簡單、在稠密圖上自然高效、始終成長為一棵連通樹。
- 缺點：在稀疏圖上要快就需要優先佇列。
- 適用：連通加權圖的最小生成樹，尤其是稠密圖。

## 小結

- 成長單一棵樹。
- 反覆加入最便宜的橫跨邊。
- 切割性質讓每一次選擇都安全。
