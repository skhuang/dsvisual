## 左傾堆積(Leftist Heap)

以 null-path-length (NPL) 為依據保持右脊最短的二元堆積,所有操作皆以 merge 為原語,提供 $O(\log N)$ 合併、插入與取出。

---

## 核心概念

節點的 NPL 定義為到達最近 null 子節點的最短路徑長度。左傾性質:對每個節點,左子節點的 NPL $\geq$ 右子節點的 NPL,確保右脊長度 $\leq \log_2(N+1)$。

- merge(a, b):較小根留下,其右子樹與另一棵堆積遞迴合併;若合併後右 NPL > 左 NPL,則交換左右子。
- insert(x):建立單節點堆積,與主堆積 merge。
- extractTop:移除根,merge 左右子樹。

---

## 運作流程

1. merge(a, b):若 a 為 null 回傳 b;若 b 為 null 回傳 a。
2. 比較 a.key 與 b.key,令 a 為鍵值較小者(min-heap)。
3. 遞迴:a.right = merge(a.right, b)。
4. 維護左傾性質:若 NPL(a.left) < NPL(a.right),交換 a 的左右子;更新 a.npl = NPL(a.right) + 1。

## 複雜度分析

| 操作 | 時間(最壞) | 空間 |
| --- | --- | --- |
| merge | $O(\log N)$ | $O(\log N)$ |
| insert | $O(\log N)$ | $O(\log N)$ |
| peek | $O(1)$ | $O(1)$ |
| extractTop | $O(\log N)$ | $O(\log N)$ |
| 空間合計 | — | $O(N)$ |

$$T_{\text{merge}}(N) = O(\log N)$$

merge 只沿右脊遞迴,右脊長度 $\leq \log_2(N+1)$,故每次 merge 最多 $O(\log N)$ 步。

## 程式碼

```cpp
LNode* mergeNodes(LNode* a, LNode* b) {
    if (!a)
        return b;
    if (!b)
        return a;
    if (!cmp(a->key, b->key))
        swap(a, b); // ensure a has smaller key

    a->right = mergeNodes(a->right, b); // recurse along right spine

    // restore leftist property
    if (getNpl(a->left) < getNpl(a->right))
        swap(a->left, a->right);
    a->npl = getNpl(a->right) + 1;
    return a;
}

void insert(int x) { root = mergeNodes(root, new LNode(x)); }

int extractTop() {
    int out = root->key;
    LNode *l = root->left, *r = root->right;
    delete root;
    root = mergeNodes(l, r);
    return out;
}
```

## 優缺點與使用時機

- 優點:$O(\log N)$ merge,適合需要頻繁合併兩個獨立堆積的場景。
- 優點:結構規律(NPL 保證右脊短),易於分析。
- 缺點:需儲存 npl 欄位,且為指標結構,快取效能不如 Binary Heap。
- 缺點:decrease-key 無法像 Fibonacci Heap 達到 $O(1)$ 攤銷。
- 適用:需要合併優先佇列的圖演算法、外部排序中的多路合併。

---

## 小結

- NPL 左傾性質保證右脊長度 $O(\log N)$,所有操作皆以 merge 表達。
- merge/insert/extractTop 皆 $O(\log N)$ 最壞;peek $O(1)$。
- 相較 Skew Heap 多一個 npl 欄位,但提供嚴格最壞保證(非攤銷)。
