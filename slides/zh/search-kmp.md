## KMP 字串比對演算法

KMP（Knuth-Morris-Pratt）演算法透過預先計算模式字串的 LPS 陣列，在發生不匹配時直接決定模式可以跳移多遠，從而避免重新比對已比對過的字元；整體時間複雜度為 $O(n+m)$。

---

## 核心概念

**LPS 陣列**（最長公共前後綴，a.k.a. failure function）記錄模式每個位置的最長真前綴同時也是後綴的長度。發生不匹配時，直接令 `j = lps[j-1]` 而不是從頭重來。

- LPS 陣列在 $O(m)$ 時間內預建完成。
- 文字指標 `i` 永不後退，每步至少前進一格。
- 總比對次數 $\le 2n$，整體複雜度 $O(n+m)$。

---

## 運作流程

1. **預處理**：對模式字串執行 `computeLPS`，建立 LPS 陣列。
2. **匹配推進**：當 `text[i] == pat[j]` 時同時令 `i++` 與 `j++`；若 `j == m` 則記錄一次成功匹配，並令 `j = lps[j-1]`。
3. **不匹配處理**：若 `j > 0`，令 `j = lps[j-1]`（不移動 `i`）；若 `j == 0`，令 `i++`。

## 複雜度分析

| 階段 | 時間 | 空間 |
| --- | --- | --- |
| 建立 LPS | $O(m)$ | $O(m)$ |
| 搜尋 | $O(n)$ | $O(1)$ |
| 總計 | $O(n+m)$ | $O(m)$ |

$$T_{\text{KMP}} = O(n + m)$$

KMP 是線性時間演算法；文字指標永不後退，因此不會重複掃描。

---

## 程式碼

```cpp
vector<int> computeLPS(const string& pat) {
    int m = pat.size();
    vector<int> lps(m, 0);
    int len = 0;
    for (int i = 1; i < m;) {
        if (pat[i] == pat[len]) {
            lps[i++] = ++len;
        } else if (len != 0) {
            len = lps[len - 1];
        } else {
            lps[i++] = 0;
        }
    }
    return lps;
}

void kmpSearch(const string& text, const string& pat) {
    int n = text.size(), m = pat.size();
    vector<int> lps = computeLPS(pat);
    int i = 0, j = 0;
    while (i < n) {
        if (text[i] == pat[j]) {
            i++; j++;
            if (j == m) {
                cout << "Match at index " << (i - j) << endl;
                j = lps[j - 1];
            }
        } else if (j != 0) {
            j = lps[j - 1];
        } else {
            i++;
        }
    }
}
```

## 優缺點與使用時機

- 優點：保證線性時間，文字指標絕不後退。
- 優點：LPS 陣列建構簡單，只需 $O(m)$ 時間與空間。
- 缺點：常數因子高於 Boyer-Moore，在一般文字上實際速度較慢。
- 缺點：未利用字母表特性，無法像 BM 那樣大幅跳移。
- 適用：最壞情況線性保證至關重要的場景（例如重複性高的字串）。

## 小結

- LPS（failure function）是 KMP 的核心預處理資料。
- 文字指標永不後退；總比對次數 $\le 2n$。
- 時間 $O(n+m)$，空間 $O(m)$。
- 與 BM、RK 的比較見 search-strcompare。
