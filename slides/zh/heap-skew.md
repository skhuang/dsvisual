## 偏斜堆積(Skew Heap)

自調整合併堆積:每次 merge 後無條件交換左右子節點,無需儲存 npl 欄位,以攤銷 $O(\log N)$ 的代價完成 merge、insert 與 extractTop。

---

## 核心概念

Skew Heap 是 Leftist Heap 的簡化版本:Leftist Heap 只在 NPL 違反時交換子節點,而 Skew Heap 每次都無條件交換,以攤銷分析保證 $O(\log N)$ 效能而不維護任何平衡資訊。

- merge(a, b):令較小根為 a;a.right = merge(a.right, b);然後無條件 swap(a.left, a.right)。
- insert(x):建立單節點堆積,與主堆積 merge。
- extractTop:移除根,merge 左右子樹。
- 無 npl 欄位,節點結構最小化。

---

## 運作流程

1. merge(a, b):若任一為 null 回傳另一方。
2. 令 a 的根鍵值 $\leq$ b 的根鍵值(若否則 swap(a, b))。
3. 遞迴:a.right = merge(a.right, b)。
4. 無條件交換 a 的左右子:swap(a.left, a.right)。回傳 a。

## 複雜度分析

| 操作 | 攤銷時間 | 空間 |
| --- | --- | --- |
| merge | $O(\log N)$ | $O(\log N)$ |
| insert | $O(\log N)$ | $O(\log N)$ |
| peek | $O(1)$ | $O(1)$ |
| extractTop | $O(\log N)$ | $O(\log N)$ |
| 空間合計 | — | $O(N)$ |

$$T_{\text{merge}} = O(\log N)\text{ amortized}$$

以重 / 輕路徑勢能函數分析:無條件交換確保整個操作序列的平均代價為 $O(\log N)$。

## 程式碼

```cpp
SNode* mergeNodes(SNode* a, SNode* b) {
    if (!a) return b;
    if (!b) return a;
    if (!cmp(a->key, b->key)) swap(a, b); // ensure a has smaller key

    a->right = mergeNodes(a->right, b);   // recurse along right
    swap(a->left, a->right);              // unconditional swap
    return a;
}

void insert(int x) {
    root = mergeNodes(root, new SNode(x));
}

int extractTop() {
    int out = root->key;
    SNode* l = root->left, *r = root->right;
    delete root;
    root = mergeNodes(l, r);
    return out;
}

void merge(SkewHeap& other) {
    root = mergeNodes(root, other.root);
    other.root = nullptr;
}
```

## 優缺點與使用時機

- 優點:比 Leftist Heap 更簡單,無需 npl 欄位,程式碼最精簡。
- 優點:攤銷 $O(\log N)$ merge,適合頻繁合併的場景。
- 缺點:單次操作最壞可能退化至 $O(N)$,不適合嚴格延遲要求的系統。
- 缺點:指標結構,快取效能不如 Binary Heap。
- 適用:快速原型、教學展示、以及需要 merge 但不需 decrease-key 的中等規模應用。

---

## 小結

- Leftist Heap 的簡化:去掉 npl,每次 merge 無條件交換子節點。
- merge/insert/extractTop 攤銷 $O(\log N)$;peek $O(1)$。
- 自調整性質使其結構隨操作序列自然平衡,無需顯式平衡資訊。
